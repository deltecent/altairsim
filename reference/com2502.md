# COM2502 / COM2017 Universal Asynchronous Receiver/Transmitter (UART)

Source: [com2502.pdf](#)

SMC Microsystems Corporation. Also marketed as COM2502/H and COM2017/H (the
"H" suffix is the higher clock-rate part). The COM2017/H additionally supports a
5-bit data code (which enables the 1.5-stop-bit mode). This is a classic
first-generation single-chip UART in a 40-pin DIP; it is the transmit/receive
engine behind the 88-SIO serial card. It is fully double-buffered on both the
transmit and receive sides, and every framing option (word length, parity mode,
stop bits, duplex) is set by hard-wired or strobed control pins rather than by
software registers.

Because framing is set by pins, an emulator normally treats the framing options
as fixed configuration for the card, and only models the byte-level data path
(holding registers) and the status flags that guest software actually polls.

---

## Programming model overview

- **Two independent sections**, receiver and transmitter, sharing ONE set of
  control (framing) bits. Full-duplex is native; the two directions may even run
  at different baud rates because each has its own 16x clock input (RCP for the
  receiver, TCP for the transmitter).
- **Framing is programmed via control pins**, not a memory-mapped register:
  data-bits (NDB1/NDB2), parity mode (NPB/POE), and stop-bits (NSB). These are
  latched through a **control bits holding register** by strobing **CS** high
  (or CS may be hard-wired high, in which case the pin levels are always live).
- **Clocks are 16x (16X) the baud rate.** RCP and TCP each run at 16 times the
  desired baud rate for their direction. Start bits are sampled/verified against
  this 16x clock.
- **Data is tri-state and bus-oriented.** The receiver data outputs (RD1-RD8)
  are enabled by **RDE** (active low); the five status outputs plus TBMT are
  enabled by **SWE** (active low). This lets the data byte and the status byte
  share a data bus and be read separately.
- **Data is right-justified.** The LSB is always on RD1 (receive) and TD1
  (transmit). For word lengths under 8 bits, the unused high data lines read low
  on receive; on transmit, unused TD inputs may be left in any logic state.

## Pin functions (programming-relevant)

Active-low signals are written here with a trailing `#` (the datasheet overlines
them). Pin numbers are exact.

### Power / reset

| Pin | Symbol | Name | Function |
|-----|--------|------|----------|
| 1 | VCC | Power | +5 V supply |
| 2 | VDD | Power | -12 V supply |
| 3 | GND | Ground | Ground |
| 21 | MR | Master Reset | Pulse HIGH after power-up. Sets TSO, TEOC, TBMT high (line marking, transmitter at rest); resets RDA, RPE, RFE, ROR low. Does NOT clear the control/framing bits. |

### Receiver

| Pin | Symbol | Name | Function |
|-----|--------|------|----------|
| 4 | RDE# | Received Data Enable | Low-level input enables the receiver buffer outputs RD8-RD1 onto the bus. |
| 5-12 | RD8-RD1 | Receiver Data Outputs | 8 tri-state data outputs (pin 5 = RD8 (MSB) ... pin 12 = RD1 (LSB)). Enabled by RDE#. Unused high lines (per word length) read low; data is right-justified (LSB on RD1). |
| 13 | RPE | Receiver Parity Error | Tri-state status output (enabled by SWE#). HIGH if the received parity bit does not agree with the selected parity. Forced/held low when no-parity mode is selected. |
| 14 | RFE | Receiver Framing Error | Tri-state status output (enabled by SWE#). HIGH if the received character has no valid stop bit. |
| 15 | ROR | Receiver Over Run | Tri-state status output (enabled by SWE#). HIGH if the previous character was not read (RDA not reset) before the present character was transferred into the receiver buffer register. |
| 16 | SWE# | Status Word Enable | Low-level input enables the five status outputs plus TBMT: RPE, RFE, ROR, RDA, and TBMT. |
| 17 | RCP | Receiver Clock | Clock at 16x (16X) the desired receiver baud rate. |
| 18 | RDAR# | Receiver Data Available Reset | Low-level input resets the RDA output low (i.e. "I have read the byte"). |
| 19 | RDA | Receiver Data Available | Tri-state status output (enabled by SWE#). HIGH when a complete character has been received and transferred into the receiver buffer register. |
| 20 | RSI | Receiver Serial Input | Serial data input. A HIGH (mark) to LOW (space) transition initiates reception (start bit). |

### Transmitter

| Pin | Symbol | Name | Function |
|-----|--------|------|----------|
| 22 | TBMT | Transmitter Buffer Empty | Tri-state status output (enabled by SWE#). HIGH when the transmitter buffer register may be loaded with new data. |
| 23 | TDS# | Transmitter Data Strobe | Low-level input strobe: loads the data bits (TD1-TD8) into the transmitter buffer register. |
| 24 | TEOC | Transmitter End Of Character | HIGH for one bit time after a full character is transmitted; stays high until the start of the next character, or for one-half of a TCP period during continuous transmission. |
| 25 | TSO | Transmitter Serial Output | Serial output. Stays HIGH (marking) when no data is being transmitted. |
| 26-33 | TD1-TD8 | Transmitter Data Inputs | 8 data inputs (pin 26 = TD1 (LSB) ... pin 33 = TD8 (MSB)), strobed in by TDS#. LSB must be on TD1; unused high lines (per word length) may be any logic state. |
| 40 | TCP | Transmitter Clock | Clock at 16x (16X) the desired transmitter baud rate. |

### Control / framing inputs

| Pin | Symbol | Name | Function |
|-----|--------|------|----------|
| 34 | CS | Control Strobe | HIGH-level input latches the control bits (NDB1, NDB2, NSB, POE, NPB) into the control-bits holding register. May be strobed or hard-wired high. |
| 35 | NPB | No Parity Bit | HIGH eliminates the parity bit (stop bit(s) follow the last data bit directly, both TX and RX); RPE is forced low. See POE (pin 39). |
| 36 | NSB | Number of Stop Bits | LOW = 1 stop bit; HIGH = 2 stop bits. HIGH with a 5-bit word length instead generates 1.5 stop bits (COM2017/H). |
| 37 | NDB2 | Number of Data Bits (bit 2) | With NDB1, selects 5/6/7/8 data bits (table below). |
| 38 | NDB1 | Number of Data Bits (bit 1) | With NDB2, selects 5/6/7/8 data bits (table below). |
| 39 | POE | Odd/Even Parity Select | With NPB, selects odd/even/no parity (table below). |

## Control word (framing) programming

There is no single 8-bit "control register"; the framing is the combination of
the five control pins latched by CS. The three programmable fields are:

### Data bits per character (NDB2, NDB1)

| NDB2 | NDB1 | Data bits/character |
|------|------|---------------------|
| L | L | 5 |
| L | H | 6 |
| H | L | 7 |
| H | H | 8 |

### Parity mode (NPB, POE)

| NPB | POE | Mode |
|-----|-----|------|
| L | L | Odd parity |
| L | H | Even parity |
| H | X | No parity (POE = don't care) |

In no-parity mode the RPE output is forced low, and the receiver expects the
stop bit(s) immediately after the last data bit.

### Stop bits (NSB)

| NSB | Stop bits |
|-----|-----------|
| L | 1 |
| H | 2 (or **1.5** when the word length is 5 data bits, COM2017/H) |

## Status word (SWE#-enabled outputs)

When SWE# is low, these outputs are driven onto the bus together. This is the
byte a guest polls to manage the serial link. (There is no chip-defined bit
position — the 88-SIO card wires these pins to specific data-bus bits; consult
the card, not the chip, for the bit map. Listed with their pin numbers.)

| Signal | Pin | Active state | Meaning | Cleared by |
|--------|-----|--------------|---------|------------|
| TBMT | 22 | HIGH | Transmitter buffer empty — OK to load a new byte via TDS#. | Goes low when a byte is strobed in (TDS#) and the shift register is busy; returns high when the buffer moves to the shift register. |
| RDA | 19 | HIGH | Receiver Data Available — a full character sits in the receiver buffer. | RDAR# (pin 18) low. |
| ROR | 15 | HIGH | Receiver Over Run — a new character overwrote one that was never read (RDA still high at transfer time). | (Reset by MR; cleared on the next non-overrun character load.) |
| RFE | 14 | HIGH | Framing error — no valid stop bit in the received character. | Updated per received character; MR resets low. |
| RPE | 13 | HIGH | Parity error — received parity disagrees with selected parity. Held low in no-parity mode. | Updated per received character; MR resets low. |

Notes:
- These status flip-flops (RPE, RFE, ROR) are updated in the **status word
  holding register** as each character completes, at the same instant RDA is
  raised — so the error flags are valid for the character currently readable via
  RDA. Read status and data as a matched pair.
- **Overrun detection:** when a full character is assembled, internal logic
  checks RDA. If RDA is still HIGH (previous byte unread), the over-run flip-flop
  is set. The ONLY way the receiver learns the byte was read is RDA being reset
  low (via RDAR#).

## Transmitter operation (holding-register behavior)

Double-buffered: a **data bits holding register (buffer)** feeds a **transmitter
shift register**.

1. After MR, with TBMT and TEOC high (line marking), set the data bits on
   TD1-TD8 and (if changing framing) strobe CS. TDS# and CS may occur
   simultaneously as long as pulse-width minimums are met.
2. Strobe **TDS#** low: the data byte is loaded into the buffer register and
   **TBMT goes low** (buffer full, cannot accept new data).
3. If the shift register is empty (or finishes the current byte), the buffer is
   transferred into the shift register and transmission begins:
   - **TSO goes low (start bit)**, **TEOC goes low**, and **TBMT goes high
     again** — the buffer is now free to be loaded with the NEXT byte.
4. Loading a new byte while the current one is still shifting drives TBMT low
   again until the present transmission completes. One full character time is
   available to load the next byte with no loss of transmission speed (this is
   the benefit of double buffering).
5. Serial frame order: **start bit, data bits (LSB first), parity bit (if
   selected), stop bit(s)**. When the last stop bit has been on the line for one
   bit time, **TEOC goes high**. If TBMT is low (a byte is waiting) transmission
   continues immediately; if TBMT is high the transmitter comes fully to rest,
   and new control (framing) bits may be loaded before the next transmission.

Start-bit placement: on data-transmission initiation (or at less than 100% line
utilization), the start bit is placed on TSO at the high-to-low transition of
the TCP clock following the trailing edge of TDS#.

## Receiver operation (holding-register behavior)

Double-buffered: a **receiver shift register** feeds a **data bits holding
register (buffer)** read out on RD1-RD8.

1. Reception begins when RSI transitions mark (high) to space (low).
2. **Start-bit verification:** if RSI remains spacing for 1/2 a bit time, the
   start bit is verified. If the line returns to marking before 1/2 bit time,
   verification restarts (false-start rejection). This is the "start bit
   verification decreases error rate" feature.
3. Data bits are then sampled at their bit centers (center-bit sampling against
   the 16x RCP clock), followed by the parity bit (if selected) and the stop
   bit(s).
4. On assembling a full character:
   - Parity check: if the received parity bit disagrees with the transmitted/
     selected parity, the parity-error flip-flop is set high. In no-parity mode
     the parity-error flip-flop is unconditionally held low.
   - Framing check: if a valid stop bit is not received, the framing-error
     flip-flop is set high.
   - Overrun check: internal logic examines RDA; if RDA is still high (prior
     character unread), the over-run flip-flop is set.
5. The character is transferred from the shift register into the data-bits
   holding register and **RDA goes high**. Because of double buffering, a full
   character time is available to read out the character before the next one
   arrives.
6. Read the byte via RDE# (data) and SWE# (status), then pulse **RDAR#** to
   reset RDA low ("byte consumed").

Receiver timing detail: after RDA is raised, if the RDA line had NOT been reset
(ROR = high level) it behaves per the "RDA*" waveform; if it HAD been reset
(ROR = low level) it behaves per the "RDA**" waveform — i.e. the RDA edge timing
relative to the last stop bit differs by whether the previous byte was read.

## Clock / baud-rate facts

- RCP (pin 17) and TCP (pin 40) are each **16x** the desired baud rate for their
  respective direction. Independent clocks allow different receive and transmit
  baud rates.
- Max clock frequency: **400 kHz** for COM2502/COM2017; **640 kHz** for the "H"
  parts (COM2502H/COM2017H). Minimum is DC (fully static).
- High-speed rating: 40K baud (implied by the 640 kHz / 16 relationship).

### Key AC timing (min pulse widths and delays)

| Parameter | Value | Pin(s) |
|-----------|-------|--------|
| Clock pulse width, min | 1 µs | RCP, TCP |
| Master reset pulse width, min | 500 ns | MR |
| Control strobe pulse width, min | 200 ns | CS |
| Transmitter data strobe width, min | 200 ns | TDS# |
| Receiver-data-available reset width, min | 200 ns | RDAR# |
| Data/control setup & hold time | 0 ns (may be 0) | TD1-TD8; NPB/NSB/NDB2/NDB1/POE |
| Strobe-to-output delay (RDE#, SWE#), max | 350 ns | receive data / status word enable |
| Output disable delay, max | 350 ns | RDE#, SWE# |
| RDAR# low width → RDA low | 200 ns pulse; RDA drops after ~300 ns | RDAR# / RDA |
| TDS# high edge → TBMT low | ~400 ns | TDS# / TBMT |

Input info (data/control) need only be valid during the last Tpw,min of the
input strobes (TDS#, CS) — setup and hold may both be 0.

## Timing/behavior quirks worth emulating or noting

1. **Start bit appears within one clock period (TCP) after the trailing edge of
   TDS#**, provided TEOC and TBMT were high (transmitter inactive) at load time.
2. **Start-bit detection latency:** the start bit (mark-to-space transition) is
   always detected within one RCP clock period — a maximum start-slippage of
   1/16 of a bit time.
3. **Tri-state OFF is controlled by RDE# and SWE#.** Each tri-state output has
   three states: low-Z to VCC, low-Z to GND, and high-Z OFF (~10 MΩ). Not more
   than one output should be shorted at a time.
4. **MR does not clear framing/control bits** — only CS loads/changes those. MR
   only resets the data-path status (RDA/RPE/RFE/ROR low) and idles the
   transmitter (TSO/TEOC/TBMT high).
5. **No-parity mode alters receiver framing expectations:** the receiver expects
   the stop bit(s) to follow the last data bit immediately, and RPE is forced
   low.
6. **Overrun is only cleared by the guest reading (RDA reset via RDAR#).** If the
   guest never pulses RDAR#, every subsequent byte sets ROR.
7. **1.5 stop bits** exist only for the 5-data-bit code on the COM2017/H
   (NSB high + 5-bit word length).
8. Under steady-state conditions no current flows for TTL or MOS interfacing
   (COM2502/H) — relevant only to electrical modeling, not logic emulation.

## Functional block summary

Transmitter path: CONTROL BITS HOLDING REGISTER + DATA BITS HOLDING REGISTER
(BUFFER) → STEERING LOGIC → TRANSMITTER SHIFT REGISTER → OUTPUT LOGIC → TSO;
with PARITY BIT GENERATION LOGIC and a TIMING GENERATOR driven by the 16xT
clock; an R/S flip-flop produces TRANSMITTER BUFFER EMPTY (TBMT) and END OF
CHARACTER (TEOC).

Receiver path: RSI → START BIT VERIFICATION → RECEIVER SHIFT REGISTER (16xR
clock timing generator) → PARITY BIT CHECKING LOGIC / RIGHT JUSTIFY LOGIC →
DATA BITS HOLDING REGISTER BUFFER → (AND gate enabled by DATA ENABLE/RDE#) →
RD1-RD8. A DATA AVAILABLE R/S flip-flop (reset by RDAR#) drives RDA, and the
STATUS WORD HOLDING REGISTER (enabled by SWE#) carries FRAMING ERROR, OVER RUN,
PARITY ERROR, DATA AVAILABLE, and TRANSMITTER BUFFER EMPTY.
