# MC6850 Asynchronous Communications Interface Adapter (ACIA)

Source: [6850.pdf](#)

Motorola MC6850 / MC68A50 (1.5 MHz) / MC68B50 (2.0 MHz). N-channel silicon-gate MOS.
The ACIA is the UART used by the MITS 88-2SIO serial board emulated by altairsim.

## Overview

The ACIA provides data formatting and control to interface serial asynchronous
data communications to a bus-organized (8-bit bidirectional) system. The bus
interface includes select, enable, read/write, interrupt, and bus interface logic.
Parallel data from the bus is serialized on transmit and deserialized on receive,
with formatting and error checking. The functional configuration is programmed via
the bus during system initialization.

### Features
- 8- and 9-bit transmission
- Optional even and odd parity
- Parity, overrun, and framing error checking
- Programmable control register
- Optional ÷1, ÷16, and ÷64 clock modes
- Up to 1.0 Mbps transmission
- False start bit deletion
- Peripheral / modem control functions
- Double buffered (transmit and receive)
- One- or two-stop-bit operation

## Registers

At the bus interface the ACIA appears as two addressable memory locations.
Internally there are four registers: two read-only and two write-only. The
register accessed is chosen by the Register Select (RS) line and the Read/Write
(R/W) line.

| RS | R/W | Register selected |
|----|-----|-------------------|
| 0  | Write (low) | Control Register (write only) |
| 0  | Read (high) | Status Register (read only) |
| 1  | Write (low) | Transmit Data Register / TDR (write only) |
| 1  | Read (high) | Receive Data Register / RDR (read only) |

- **Transmit Data Register (TDR):** Data is written into the TDR on the negative
  transition of E when addressed with RS high and R/W low. Transferred to the
  transmit Shift Register within 1 bit-time of the write if the transmitter is idle,
  otherwise when the current character completes. The transfer sets TDRE.
- **Receive Data Register (RDR):** A completed received character is automatically
  transferred from the receive deserializer into the empty RDR. This sets RDRF.
  Reading the RDR (address ACIA, RS high, R/W high) clears RDRF (non-destructive
  read; status retained by RDRF). While RDR is full, automatic transfer from the
  shift register is inhibited so the character remains valid.

### Table 1 — ACIA Register Contents (per data-bus bit)

| Bit | Transmit Data Register (RS·R/W = write) | Receive Data Register (RS·R/W = read) | Control Register (write only) | Status Register (read only) |
|-----|------------------------------------------|----------------------------------------|-------------------------------|------------------------------|
| 0 | Data Bit 0* (leading bit, LSB) | Data Bit 0 | Counter Divide Select 1 (CR0) | Receive Data Register Full (RDRF) |
| 1 | Data Bit 1 | Data Bit 1 | Counter Divide Select 2 (CR1) | Transmit Data Register Empty (TDRE) |
| 2 | Data Bit 2 | Data Bit 2 | Word Select 1 (CR2) | Data Carrier Detect (DCD) |
| 3 | Data Bit 3 | Data Bit 3 | Word Select 2 (CR3) | Clear-to-Send (CTS) |
| 4 | Data Bit 4 | Data Bit 4 | Word Select 3 (CR4) | Framing Error (FE) |
| 5 | Data Bit 5 | Data Bit 5 | Transmit Control 1 (CR5) | Receiver Overrun (OVRN) |
| 6 | Data Bit 6 | Data Bit 6 | Transmit Control 2 (CR6) | Parity Error (PE) |
| 7 | Data Bit 7*** | Data Bit 7** | Receive Interrupt Enable (CR7) | Interrupt Request (IRQ) |

\* Leading bit = LSB = bit 0.
\*\* Data bit 7 will be zero in 7-bit-plus-parity modes.
\*\*\* Data bit 7 is "don't care" in 7-bit-plus-parity modes.

## Control Register (write only, RS=0, R/W=low)

Eight bits of write-only buffer. Controls function of receiver, transmitter,
interrupt enables, and the RTS peripheral/modem output.

### Counter Divide Select — CR1, CR0

Determine the clock divide ratio applied to the Tx and Rx clocks. The `÷1`, `÷16`,
`÷64` ratios divide the external Tx CLK / Rx CLK down to the bit rate. **CR1 CR0 = 1 1
is a Master Reset**, not a divide ratio.

| CR1 | CR0 | Function |
|-----|-----|----------|
| 0 | 0 | ÷1 |
| 0 | 1 | ÷16 |
| 1 | 0 | ÷64 |
| 1 | 1 | Master Reset |

### Word Select — CR4, CR3, CR2

Selects word length, parity, and number of stop bits. Word length, parity, and
stop-bit changes are **not buffered** and become effective immediately.

| CR4 | CR3 | CR2 | Function |
|-----|-----|-----|----------|
| 0 | 0 | 0 | 7 bits + Even Parity + 2 Stop Bits |
| 0 | 0 | 1 | 7 bits + Odd Parity + 2 Stop Bits |
| 0 | 1 | 0 | 7 bits + Even Parity + 1 Stop Bit |
| 0 | 1 | 1 | 7 bits + Odd Parity + 1 Stop Bit |
| 1 | 0 | 0 | 8 bits + 2 Stop Bits |
| 1 | 0 | 1 | 8 bits + 1 Stop Bit |
| 1 | 1 | 0 | 8 bits + Even Parity + 1 Stop Bit |
| 1 | 1 | 1 | 8 bits + Odd Parity + 1 Stop Bit |

### Transmitter Control — CR6, CR5

Control the RTS output, the transmit (TDRE) interrupt, and transmission of a Break
(space) level.

| CR6 | CR5 | Function |
|-----|-----|----------|
| 0 | 0 | RTS = low, Transmitting Interrupt Disabled |
| 0 | 1 | RTS = low, Transmitting Interrupt Enabled |
| 1 | 0 | RTS = high, Transmitting Interrupt Disabled |
| 1 | 1 | RTS = low, Transmits a Break level on the Transmit Data output. Transmitting Interrupt Disabled |

### Receive Interrupt Enable — CR7

When set (1), IRQ is enabled for the following receiver conditions:
Receive Data Register Full (RDRF), Overrun, and a low-to-high transition on the
DCD input.

## Status Register (read only, RS=0, R/W=high)

| Bit | Name | Meaning when set (1) |
|-----|------|----------------------|
| 0 | RDRF — Receive Data Register Full | Received data has been transferred to the RDR. Cleared by reading the RDR or by Master Reset. DCD high also forces RDRF to indicate empty. |
| 1 | TDRE — Transmit Data Register Empty | TDR is empty; new data may be written. Low = register full, or CTS is high (CTS high inhibits/holds TDRE low). |
| 2 | DCD — Data Carrier Detect | Reflects the DCD input from a modem. A low-to-high DCD transition sets this bit and (if enabled) causes an interrupt. Stays high after DCD returns low until the status register is read and then the RDR is read. |
| 3 | CTS — Clear-to-Send | Reflects the CTS input. High CTS inhibits TDRE (holds it low). Not affected by Master Reset. |
| 4 | FE — Framing Error | Received character improperly framed — no stop bit detected. Present throughout the time the associated character is available. Also set by a Break. |
| 5 | OVRN — Receiver Overrun | One or more characters were lost: a character was received but the previous character had not been read from the RDR before the next was transferred. Set at the midpoint of the last bit of the character preceding the one that was read (i.e., appears when the valid character prior to overrun is read). RDRF remains set until overrun is reset. Reset by reading the RDR or by Master Reset. |
| 6 | PE — Parity Error | Number of ones in the character does not match preselected odd/even parity. Present as long as the data character is in the RDR. If no parity selected, both transmit parity generation and receive parity checking are inhibited. |
| 7 | IRQ — Interrupt Request | State of the IRQ output. Any enabled interrupt condition sets it. When IRQ output is low (active), this bit reads high. Cleared by reading the RDR or writing the TDR. |

## Power-On / Master Reset

- Master Reset is issued by writing CR1 CR0 = 1 1 to the Control Register. It must
  be applied during system initialization before programming the ACIA config.
- During Master Reset, IRQ and RTS outputs are held at level 1 (high).
- On the first Master Reset, RTS output can be programmed high or low with the IRQ
  output held high — CR5 and CR6 should also be programmed to define the RTS state.
- Internal power-on reset logic detects the power-turn-on transition and holds the
  chip in a reset state to prevent erroneous output transitions prior to
  initialization; released by the bus-programmed Master Reset.
- After Master Reset, program the divide ratio (CR0/CR1), word format
  (CR2–CR4), and interrupt/RTS controls (CR5–CR7). After reset the clock divide
  bits must be set high (CR1 CR0 = 1 1) before a new divide ratio is selected —
  note: after power-on or a power-fail restart, Master Reset must occur first.

## Clock Inputs (Tx CLK / Rx CLK)

- Separate high-impedance TTL-compatible clock inputs for transmit and receive.
- Selectable frequencies of 1, 16, or 64 times the data rate (see CR0/CR1).
- **Transmit Clock (Tx CLK):** clocks transmitted data; data is initiated on the
  **negative** transition of the clock.
- **Receive Clock (Rx CLK):** synchronizes received data. In ÷1 mode the clock must
  be synchronized to the data (externally). The receiver samples data on the
  **positive** transition of the clock.
- Receive bit synchronization in the ÷16 and ÷64 modes is initiated by detecting
  8 or 32 samples on the receive line (÷16 → 8 samples, ÷64 → 32 samples).
- **False start bit deletion:** in ÷16 and ÷64 modes a start bit must be present
  for at least half a bit time or it is rejected (ensures a valid start).

### Bus interface signals
- **D0–D7 — Bidirectional Data:** three-state; high-impedance (off) except during an
  ACIA read.
- **E — Enable:** high-impedance TTL input that enables the bus I/O buffers and clocks
  data to/from the ACIA. Derived from the MC6800 φ2 clock. Data is written to the TDR
  on the **negative transition of E**.
- **R/W — Read/Write:** high = MPU read (ACIA drives the bus with the selected
  read-only register); low = MPU write (data into a selected write-only register).
- **Chip Select — CS0, CS1, CS2:** three high-impedance TTL address lines. The ACIA is
  selected when **CS0 high AND CS1 high AND CS2 low**. Transfers occur under control of
  E, R/W, and RS.
- **RS — Register Select:** high selects the Transmit/Receive Data Registers; low
  selects the Control/Status Registers. Used with R/W to pick the read-only vs
  write-only register of each pair.
- **IRQ — Interrupt Request:** TTL-compatible, open-drain (no internal pullup),
  active-low output. Goes low when an enabled interrupt condition is present and the
  IRQ status bit is set. Remains low until the interrupt source is cleared.

### Serial / modem control signals
- **Rx Data — Receive Data:** high-impedance TTL input, serial data in. In ÷16/÷64
  modes the internal clock is used for bit synchronization.
- **Tx Data — Transmit Data:** serial data output to a modem/peripheral.
- **CTS — Clear-to-Send:** high-impedance TTL input, active low. Provides automatic
  control of the transmitter via the modem CTS; a high CTS inhibits TDRE (holds the
  TDRE status bit low, blocking transmit).
- **RTS — Request-to-Send:** output enabling MPU control of a peripheral/modem via the
  data bus. Controlled by CR5/CR6: CR6=0 → RTS low (active); CR6=1, CR5=0 → RTS high
  (inactive). Can also serve as Data Terminal Ready (DTR).
- **DCD — Data Carrier Detect:** high-impedance TTL input, active low. A high DCD
  inhibits and initializes the receiver. A low-to-high transition can generate an
  interrupt (if enabled) to flag loss of carrier. Rx CLK must be running for proper
  DCD operation.

## Operation Notes / Quirks

### Transmit
- Typical sequence: read the Status Register (as an interrupt result or by polling)
  to check TDRE. When TDRE is high, write a character to the TDR. The character is
  moved to the transmit Shift Register, serialized, preceded by a start bit and
  followed by one or two stop bits.
- Internal parity (odd or even) can be optionally added; it occurs between the last
  data bit and the first stop bit.
- Double buffered: a second character can be loaded into the TDR while the first is
  still being shifted out; it auto-transfers to the shift register when the first
  completes.

### Receive
- Data received bit-by-bit; a divide-by-one clock is for externally synchronized
  data, ÷16 and ÷64 for internal synchronization.
- When a full character arrives it is placed in the RDR and RDRF is set. Parity (if
  enabled) is checked and error status made available with the character. On a 7-bit
  + parity mode the parity bit is stripped (D7 = 0) so only the 7 data bits are
  transferred to the MPU.
- RDRF, OVRN, PE, FE apply to the character in the RDR.

### Interrupts
- Transmitter interrupt (TDRE high) is enabled by CR6·CR5 = 0 1. It is masked by
  disabling via CR5/CR6 or by loss of CTS (CTS high inhibits the TDRE status bit).
  Cleared by writing data to the TDR or resetting.
- Receiver interrupt is enabled by CR7 (Receive Interrupt Enable) and occurs on
  RDRF high (character received), Overrun, or DCD going high (loss of carrier).
  Cleared by reading the Status Register then reading the RDR, or by resetting the
  interrupting condition. The receiver interrupt is masked by resetting CR7.
